顧問先管理 ステージB 指示書
「顧問先情報登録」と「進捗管理（議事録含む）」をUIで2画面に分離する
================================================================

ホスト = AI-OCR(仕訳作成)アプリ。komon モジュールは統合済み(iframe隔離方式)。
ステージA でデータは既に分離済み（komon/ と shinchoku/）。
本ステージBは「UIの出し分け」だけで、データ移行は不要。


----------------------------------------------------------------
0. ゴール
----------------------------------------------------------------
- ランチャー/アプリ切替のカードを2枚にする:
    顧問先情報登録      → /komon      （view='komon'）
    進捗管理(議事録含む) → /shinchoku  （view='shinchoku'）
- 中身は同じ komon モジュール(KomonApp)を、表示モードで出し分ける。


----------------------------------------------------------------
1. 変更内容（komonセッションが実装）
----------------------------------------------------------------
(1) KomonApp.tsx
    - prop を追加: view?: 'komon' | 'shinchoku'（未指定は従来どおり全部表示＝後方互換）。
    - iframe 起動前に window.__komonView = view を注入する
      （現在 __komonCore を contentWindow にセットしているのと同じ箇所で同様に）。
    - <ModuleSwitcher currentKey={view === 'shinchoku' ? 'shinchoku' : 'komon'} /> を渡す。

(2) 埋め込みアプリ（index.html → embedded.ts）
    - 起動時に window.__komonView を読み、表示を出し分ける:
        'komon'     → 顧問先情報 関連のセクション/ナビのみ表示（進捗・議事録は隠す）
        'shinchoku' → 進捗管理＋議事録 のみ表示（顧問先情報は隠す）
        未指定/不明  → 従来どおり全部表示（後方互換）
    - ナビ（タブ）も、その view に属する項目だけ出す。
    - 進捗側で顧問先一覧が必要なら、これまで通り komon/clients を読み取り参照してよい
      （同一アプリ内・読み取りのみ。書き込みは各サブツリーへ）。

(3) embedded.ts を tools/build-komon-module.mjs で再生成。
    クライアント専用・@/core利用・自前のFirebase初期化なし、は維持。


----------------------------------------------------------------
2. データ／パス（変更しない）
----------------------------------------------------------------
- 顧問先情報      → rooms/{roomKey}/komon/…
- 進捗管理＋議事録 → rooms/{roomKey}/shinchoku/…
- roomKey=SHA-256(合言葉)、共通キー "suite-room-passphrase"。Firebaseルール変更不要。


----------------------------------------------------------------
3. 納品
----------------------------------------------------------------
- 更新した src/modules/komon/ 一式（KomonApp.tsx, embedded.ts ほか）を
  ZIP にまとめて依頼者へ渡す（ディレクトリ構成 src/modules/komon/... を維持）。
- ホスト側で次を結線する（ホスト担当が実施）:
    ・src/app/komon/page.tsx     → <KomonApp view="komon" />
    ・src/app/shinchoku/page.tsx → <KomonApp view="shinchoku" />（新規）
    ・src/core/registry.ts の shinchoku を status:'ready' に切替


----------------------------------------------------------------
4. 注意
----------------------------------------------------------------
- 後方互換（view未指定で全部表示）を必ず残す。
- 合言葉→roomKey(SHA-256)・共通キーは他モジュールと一致のまま（変更しない）。
- 静的書き出し・クライアント専用を維持。embedded.ts は手編集せず再生成で更新。
