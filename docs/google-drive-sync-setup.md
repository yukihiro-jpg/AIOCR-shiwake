# Google Drive 同期 セットアップ手順

事務所内で 2 台以上の PC から「顧問先一覧」「学習パターン」「科目マスタ・税率マスタ」「進捗表」を共有するために、Google Drive を介して同期します。本書はその準備手順です。

所要時間：15〜25 分。Google Workspace のドメイン管理者権限（または OAuth クライアント作成権限）がある人が作業してください。

---

## 全体像

```
[yukihiro@ の PC] ─┐
                  ├─→ Google Drive 上の「事務所アプリ共有」フォルダ ←─→ 自動同期
[ayako@ の PC]   ─┘
```

- Google Cloud Console で「OAuth クライアント ID」を 1 つ作る（事務所で 1 個でよい）
- Google Drive に共有フォルダを 1 つ作る（事務所で 1 個でよい）
- アプリの `.env` にその 2 つの ID を書く
- 各 PC でアプリ起動時に「Google でログイン」を 1 回押す

---

## ステップ 1: Google Cloud Console で OAuth クライアント ID を作る

### 1-1. プロジェクトを作る（初回のみ）

1. https://console.cloud.google.com/ にアクセスし、yukihiro@ でログイン
2. 画面上部の「プロジェクトを選択」→「新しいプロジェクト」
3. プロジェクト名：`accounting-app-share`（任意）
4. 「作成」→ 作成完了後、画面上部のプロジェクト名がそれになっていることを確認

### 1-2. Drive API を有効化

1. 左側メニュー →「API とサービス」→「ライブラリ」
2. 検索ボックスに `Google Drive API` と入力 → クリック
3. 「有効にする」ボタンを押す

### 1-3. OAuth 同意画面を設定

1. 左側メニュー →「API とサービス」→「OAuth 同意画面」
2. User Type：
   - **Google Workspace の同一ドメイン内で使う場合** → 「内部」を選択（推奨）
   - 個人 Gmail の場合 → 「外部」を選択（後で yukihiro / ayako を「テストユーザー」に登録する必要あり）
3. 「作成」
4. アプリ名：`事務所会計アプリ`（任意）
5. ユーザー サポートメール：yukihiro@ のメールアドレス
6. デベロッパーの連絡先情報：同上
7. 「保存して次へ」
8. スコープ：「スコープを追加または削除」→ `.../auth/drive.file` を検索してチェック → 「更新」
   - これは「このアプリが作成したファイルだけ触れる」最小権限
9. 「保存して次へ」→ 「保存して次へ」→ 完了

### 1-4. OAuth クライアント ID を作成

1. 左側メニュー →「API とサービス」→「認証情報」
2. 「+ 認証情報を作成」→「OAuth クライアント ID」
3. アプリケーションの種類：**ウェブ アプリケーション**
4. 名前：`accounting-app-web`（任意）
5. **承認済みの JavaScript 生成元** に以下を追加：
   - `http://localhost:3000`（開発時）
   - 実際にアプリをデプロイしているなら、そのドメイン（例：`https://accounting.example.com`）
6. **承認済みのリダイレクト URI** は今回の方式（GIS：Google Identity Services）では不要なので空欄でOK
7. 「作成」を押すと、「クライアント ID」が表示される
   - 例：`123456789-abcdefg.apps.googleusercontent.com`
   - **これをメモ**（後で `.env` に貼る）

---

## ステップ 2: Google Drive に共有フォルダを作る

1. https://drive.google.com/ に yukihiro@ でログイン
2. 左上「＋新規」→「フォルダ」→ フォルダ名：`事務所アプリ共有`（任意）
3. 作成したフォルダを右クリック →「共有」
4. 「ユーザーとグループの追加」に `ayako@yourdomain.jp` を入力
5. 権限を「**編集者**」に設定 → 「送信」
6. もう一度フォルダを右クリック →「リンクをコピー」
   - URL は次のような形：`https://drive.google.com/drive/folders/1AbCdEfGhIjKlMnOpQrStUvWxYz`
   - 末尾の `1AbCd...` の部分が **フォルダ ID**。**これをメモ**

---

## ステップ 3: アプリの `.env` に値を貼る

プロジェクトのルートにある `.env.local`（無ければ作成）に以下を追加：

```
NEXT_PUBLIC_GOOGLE_CLIENT_ID=123456789-abcdefg.apps.googleusercontent.com
NEXT_PUBLIC_GOOGLE_DRIVE_FOLDER_ID=1AbCdEfGhIjKlMnOpQrStUvWxYz
```

- 値はステップ 1-4 と 2 でメモしたものに置き換える
- 開発サーバーが動いていれば `Ctrl+C` で止めて `npm run dev` で再起動

---

## ステップ 4: 各 PC で初回ログイン

1. アプリを開くと、画面右上に「Google でログイン」ボタンが出ます（実装後）
2. yukihiro@ の PC：ボタンを押して yukihiro@ で認証 → 「許可」
3. ayako@ の PC：同様に ayako@ で認証 → 「許可」
4. 以降は自動でログインが維持され、データが Drive 経由で共有されます

---

## トラブルシューティング

- **「このアプリは確認されていません」と出る**
  - 「内部」ユーザータイプにしている場合は出ない。「外部」の場合は「詳細」→「(アプリ名) に移動」で進める
- **「アクセスがブロックされました」と出る**
  - OAuth 同意画面で「テストユーザー」に追加されていない可能性。ステップ 1-3 で yukihiro@・ayako@ を追加
- **彩子さんがログインしてもデータが見えない**
  - Drive の共有フォルダで彩子さんが「編集者」になっているか再確認

---

## 共有しないデータ

以下は今回の同期対象外です：
- 過去の仕訳データ本体（各 PC のローカルに残る）
- 一時的なアップロード履歴

「進捗表」「顧問先」「学習パターン」「科目／税率マスタ」のみが Drive 経由で共有されます。
